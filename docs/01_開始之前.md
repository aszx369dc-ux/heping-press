# 01 開始之前

這份文件是給第一次接手這個網站的人看的。

如果您完全沒有寫過程式，也沒關係。請照順序做，一步一步完成就可以。

## 需要安裝哪些軟體

### 1. Google Chrome

用途：

- 本機打開網站測試
- 做 Hard Refresh
- 檢查圖片、翻頁、目錄、封面封底是否正常

### 2. VS Code

用途：

- 打開整個專案資料夾
- 修改 `README.md`、`data.js`、`docs/`
- 看 AI 幫您改了哪些檔案

### 3. Git

用途：

- 記錄修改
- 上傳到 GitHub
- 查看改了什麼

### 4. GitHub Desktop

用途：

- 不一定要會很多指令，也能做 Commit 與 Push
- 適合初學者先上手

### 5. Node.js

用途：

- 執行 `tools/convert-book.js`
- 把 PDF 轉成網站閱讀器要用的 PNG 圖片

### 6. Poppler / `pdftoppm`

用途：

- 真正把 PDF 每一頁轉成圖片
- `convert-book.js` 會呼叫這個工具

### 7. ChatGPT

用途：

- 幫忙整理需求
- 幫忙看 `git diff`
- 幫忙解釋錯誤訊息
- 幫忙寫給 Codex 或 Gemini 的指令

## 第一次接手完整流程

### 第 1 步：安裝 Google Chrome

先安裝 Chrome，之後本機測試網站會用到。

### 第 2 步：安裝 VS Code

安裝完成後，以後都用 VS Code 打開這個專案資料夾。

### 第 3 步：安裝 Git

安裝完成後，Windows 使用者通常會看到：

- Git Bash
- 終端機可以使用 `git` 指令

### 第 4 步：安裝 GitHub Desktop

這一步不是絕對必要，但非常建議新手安裝。

因為它可以幫您：

- Clone 專案
- 看哪些檔案被改過
- 做 Commit
- 做 Push

### 第 5 步：安裝 Node.js

安裝完成後，可以在終端機輸入：

```bash
node -v
npm -v
```

如果有顯示版本號，就代表安裝成功。

### 第 6 步：安裝 Poppler / `pdftoppm`

這是 PDF 轉圖片會用到的工具。

Windows 使用者通常要另外安裝。

安裝完成後，在終端機輸入：

```bash
pdftoppm -v
```

如果有顯示版本號，就代表安裝成功。

### 第 7 步：登入自己的 GitHub 帳號

請先確認自己有 GitHub 帳號，並已登入：

- GitHub 網站
- GitHub Desktop

### 第 8 步：接受 Repository collaborator 邀請

如果您是新加入的維護者，專案擁有者需要先邀請您成為 collaborator。

您要做的是：

1. 打開 GitHub
2. 找到邀請通知
3. 按下接受

### 第 9 步：使用 GitHub Desktop Clone Repository

在 GitHub Desktop：

1. 選 `Clone repository`
2. 選 `heping-press`
3. 選一個自己找得到的位置
4. 完成下載

### 第 10 步：用 VS Code 開啟專案

請在 VS Code 選：

`File` → `Open Folder`

然後打開剛剛 clone 下來的 `heping-press` 資料夾。

### 第 11 步：在 Terminal 檢查工具

請在 VS Code 的 Terminal 輸入：

```bash
git --version
node -v
npm -v
pdftoppm -v
```

如果四個都有正常顯示版本號，代表環境大致完成。

### 第 12 步：使用 Live Server 開啟 `index.html`

Windows 使用者可先在 VS Code 安裝 `Live Server` 擴充套件。

之後：

1. 打開 `index.html`
2. 按右下角或右鍵的 `Open with Live Server`

如果不想裝 Live Server，也可以先直接用 Chrome 打開 `index.html`。

### 第 13 步：完成一次小修改、Commit、Push 與正式網站驗收

第一次接手時，建議先做一個很小的修改練習。

例如：

- 修改 `README.md` 的一個句子
- 在 `docs/` 裡補一行說明

然後練習：

1. `git status`
2. `git add README.md`
3. `git commit -m "測試第一次交接流程"`
4. `git push`
5. 到 GitHub 確認修改已上傳

如果是網站內容檔，還要再看 GitHub Pages 是否更新成功。

## Windows 使用者常見提醒

- 看到 PowerShell、命令提示字元、Git Bash 都不用太緊張，重點是指令能正常執行
- `node -v`、`git --version`、`pdftoppm -v` 都看得到版本號才算完成
- 如果某個指令顯示「找不到」，請不要硬試更多危險指令，先把錯誤訊息貼給 AI
