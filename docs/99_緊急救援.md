# 99 緊急救援

這份文件是給完全新手用的緊急處理手冊。

每一題都先做最安全的第一步，不要急著亂試。

## 絕對不要做的事

- 不要 `git reset --hard`
- 不要 force push
- 不要刪除 `.git`
- 不要分享密碼、驗證碼或 Token

## 1. 網頁全白

### 可能原因

- `script.js` 有錯
- `data.js` 格式錯了
- 少逗號、少引號、少括號

### 最安全的第一步

先打開瀏覽器 Console，看有沒有紅字錯誤。

### 要貼給 AI 的資料

- Console 錯誤訊息
- 這次的 `git diff`

### 哪些危險指令不要做

- 不要 `git reset --hard`

## 2. 圖片出不來

### 可能原因

- 路徑打錯
- 檔名打錯
- 圖片沒放到正確資料夾

### 最安全的第一步

先確認圖片檔是否真的存在。

### 要貼給 AI 的資料

- 圖片完整路徑
- `data.js` 對應設定

### 哪些危險指令不要做

- 不要先亂刪整個資料夾

## 3. 封面封底不出現

### 可能原因

- `cover.png` 或 `back-cover.png` 不存在
- `data.js` 路徑錯
- 這本書未啟用封面封底邏輯

### 最安全的第一步

先確認圖片檔真的存在，而且檔名完全一致。

### 要貼給 AI 的資料

- 圖片路徑
- `data.js` 那本書的設定
- `script.js` 與封面封底有關的 diff

### 哪些危險指令不要做

- 不要亂改全部頁碼

## 4. 目錄跳錯頁

### 可能原因

- `startPage` 設錯
- PDF 頁數有改，後面故事沒有重算

### 最安全的第一步

先比對實際頁面和目錄。

### 要貼給 AI 的資料

- 那本書的 stories 清單
- 實際頁碼

### 哪些危險指令不要做

- 不要一次重寫整個 `data.js`

## 5. 精選星號不出現

### 可能原因

- `featured` 沒設成 `true`
- `script.js` 顯示邏輯有條件判斷

### 最安全的第一步

先看 `data.js` 那篇故事的 `featured` 值。

### 要貼給 AI 的資料

- 那篇故事的資料
- 目前畫面

### 哪些危險指令不要做

- 不要順手改其他書的設定

## 6. node 找不到

### 可能原因

- Node.js 沒安裝
- 安裝了但未重新開終端機

### 最安全的第一步

在終端機輸入：

```bash
node -v
```

### 要貼給 AI 的資料

- 完整錯誤訊息
- 作業系統版本

### 哪些危險指令不要做

- 不要亂貼不明安裝腳本

## 7. pdftoppm 找不到

### 可能原因

- Poppler 沒安裝
- 環境變數沒設好

### 最安全的第一步

在終端機輸入：

```bash
pdftoppm -v
```

### 要貼給 AI 的資料

- 錯誤訊息
- 作業系統版本

### 哪些危險指令不要做

- 不要亂刪系統資料夾

## 8. git push 失敗

### 可能原因

- 網路問題
- GitHub 登入狀態失效
- 權限不足

### 最安全的第一步

先把完整錯誤訊息複製下來。

### 要貼給 AI 的資料

- `git push` 完整錯誤文字
- `git status`

### 哪些危險指令不要做

- 不要直接 force push

## 9. GitHub Pages 一直 Queued

### 可能原因

- GitHub 伺服器暫時忙碌
- Actions 排隊中

### 最安全的第一步

先等 5 到 10 分鐘再重新整理。

### 要貼給 AI 的資料

- Actions 畫面截圖
- Pages 設定截圖

### 哪些危險指令不要做

- 不要連續亂 push 很多次

## 10. 本機有更新、正式網站沒有

### 可能原因

- 沒有 push
- 瀏覽器快取
- Actions 還沒跑完

### 最安全的第一步

先做 Hard Refresh，再看 `git status`。

### 要貼給 AI 的資料

- `git status`
- `git diff`
- 正式網站畫面

### 哪些危險指令不要做

- 不要先重裝整個專案

## 11. 不小心一次加入太多檔案

### 可能原因

- 把 PDF、ZIP、測試檔也一起加入了

### 最安全的第一步

先執行：

```bash
git status
```

看清楚哪些檔案被加入了。

### 要貼給 AI 的資料

- `git status`

### 哪些危險指令不要做

- 不要 `git reset --hard`

## 12. 不小心改壞 `script.js`

### 可能原因

- 改到不該改的邏輯
- AI 改太多

### 最安全的第一步

先看：

```bash
git diff script.js
```

### 要貼給 AI 的資料

- `git diff script.js`
- 畫面上出現的問題

### 哪些危險指令不要做

- 不要直接整份覆蓋成別的版本

## 13. 不知道要不要刪資料夾

### 可能原因

- 看起來像舊資料，但不確定是否還有引用

### 最安全的第一步

先搜尋專案有沒有地方引用這個資料夾。

### 要貼給 AI 的資料

- 資料夾名稱
- 搜尋結果

### 哪些危險指令不要做

- 不要直接刪整包資產

## 14. 不知道怎麼向 AI 求助

### 可能原因

- 不知道怎麼描述需求
- 一次講太多事情

### 最安全的第一步

請直接用這個模板：

```text
請只修改 [檔名]。
不要修改 [其他檔名]。
不要重寫整支檔案。

目標：
[描述要做的事]

先不要 Commit。
不要 Push。
完成後請提供 git diff。
```

### 要貼給 AI 的資料

- 錯誤訊息
- 相關檔案片段
- `git diff`

### 哪些危險指令不要做

- 不要把密碼、驗證碼、Token 貼給 AI
